Social share

By using correct social sharer links, you can modify the default behavior to show a popup with the sharer.

In case the JS is disabled, the user navigates to the url of the sharer.

Here is the list of the social sharer links.

Share this page on facebook or Share this page on twitter

Some browsers have already a native Share API implemented. You can have a button (or a link) that opens a native share dialog on mobile browsers.

Note: As only mobile browsers currently support that feature, you should hide the button on desktop browsers.

JS Breakpoint

Sometimes you need to listen on breakpoint changes in JS. With this helper you can do that. It will output Bootstrap breakpoints! (Try resizing your window). See how it's used in scr/demo/index.ts.

Slider (using Swiper)

By clicking the button below, we inject slider specific HTML in the page. The Async module loader will automatically see the change and load the Slider module.

By using MutationObserver you can forget about manually importing specific modules on specific pages. The loader will see a change in the subtree and automatically see if the module must be loaded or not. This way we can reduce requests to the server thus optimizing performance.

Note: if you want to support older browsers (IE10 and below), this won't work as the MutationObserver is not supported there. You can load the modules manually by enabling them in the webpack.common.js.

Bottom navbar on mobile

As most smartphone users use their devices with one hand, with the bottom navbar they can navigate your site more easily as their thumb can navigate the menu faster. This module (by default) puts the navbar at the bottom of the page until desktop. You can also pass a parameter to change the breakpoints where the bottom bar should be used. You can always disable this, by not using the "is-bottom-mobile" class.

If you don't use a fixed navbar, this module will not initialize.
